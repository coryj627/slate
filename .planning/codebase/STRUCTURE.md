## Directory Purposes

**`apps/slate-mac/Sources/SlateMac/`:**
- Purpose: All Swift source for the macOS app
- Contains: SwiftUI views, AppState, panel views, sheet views, store types, generated FFI Swift file
- Key files:
  - `SlateMacApp.swift` — `@main` entry, RootView router
  - `AppState.swift` — central @MainActor state (~4500 lines)
  - `MainSplitView.swift` — 3-column NavigationSplitView
  - `slate_uniffi.swift` — generated by uniffi-bindgen; do not edit

**`apps/slate-mac/Sources/slate_uniffiFFI/`:**
- Purpose: SwiftPM `systemLibrary` wrapper around the generated C header
- Contains: `module.modulemap`, `slate_uniffiFFI.h` (generated)
- Generated: Yes (header); `module.modulemap` is committed

**`apps/slate-mac/Tests/SlateMacTests/`:**
- Purpose: Swift unit and integration tests
- Contains: One test file per major view/model/store + milestone integration tests
- Key files: `AppStateTests.swift`, `MilestoneQIntegrationTests.swift`, `APCAContrast.swift`

**`crates/slate-core/src/`:**
- Purpose: All domain logic — parsing, indexing, querying, rendering
- Contains:
  - `session.rs` — `VaultSession` orchestrator; `SessionConfig`, `CancelToken`, paging types
  - `vault/` — `VaultProvider` trait + `FsVaultProvider` (atomic writes, trash delete)
  - `db.rs` — SQLite lifecycle, PRAGMA defaults, migration runner
  - Parser modules: `links.rs`, `tasks.rs`, `blocks.rs`, `frontmatter.rs`, `citations.rs`, `embeds.rs`, `templates.rs`, `math.rs`, `code.rs`, `diagram.rs`
  - DB query modules: `links_db.rs`, `tasks_db.rs`, `blocks_db.rs`, `citations_db.rs`, `properties_db.rs`, `search_db.rs`
  - `oplog.rs` — operation log (append-only per-file edit history)
  - `commands.rs` — `CommandRegistry` / `Command` types (used by palette)
  - `link_resolver.rs` — wikilink resolution logic + `VaultIndex` trait

**`crates/slate-core/migrations/`:**
- Purpose: Append-only SQL migration files, run by `db::open_database`
- Contains: `001_init.sql` through `013_citations.sql`
- Pattern: Never modify existing files. New migrations get the next number.

**`crates/slate-uniffi/src/`:**
- Purpose: uniffi bridge — type mirrors only, no domain logic
- Contains: `lib.rs` with all uniffi-annotated types, `VaultSession` wrapper, `ScanProgressListener` callback trait
- Key constraint: `From<core::X>` impls map types; no logic beyond mapping and argument validation (`is_allowed_status_char`)

**`docs/plans/`:**
- Purpose: Architecture decision records and milestone plans
- Key file: `05_locked_architecture_decisions.md` — canonical architecture reference

**`scripts/`:**
- Purpose: Build chain automation
- Key file: `build-mac-app.sh` — chains `cargo build` → `uniffi-bindgen` → SwiftPM build

## Key File Locations

**Entry Points:**
- `apps/slate-mac/Sources/SlateMac/SlateMacApp.swift` — `@main` Swift app entry
- `crates/slate-core/src/lib.rs` — Rust crate root; all public exports
- `crates/slate-uniffi/src/lib.rs` — FFI surface; `VaultSession::open_filesystem` is the session constructor

**Configuration:**
- `Cargo.toml` — Workspace manifest; workspace-level dependency versions
- `rust-toolchain.toml` — Pinned stable Rust version (≥ 1.89)
- `apps/slate-mac/Package.swift` — SwiftPM package; dependencies (LaTeXSwiftUI, SwiftDraw) and linker settings
- `.github/workflows/` — CI pipelines

**Core Logic:**
- `crates/slate-core/src/session.rs` — `VaultSession` struct + all vault operation implementations (~1000+ lines)
- `apps/slate-mac/Sources/SlateMac/AppState.swift` — all async orchestration, state machine (~4500 lines)
- `crates/slate-core/src/db.rs` — SQLite bootstrap and migrations

**Testing:**
- `apps/slate-mac/Tests/SlateMacTests/` — Swift tests (XCTest)
- `crates/slate-core/src/session/tests/` — Rust integration tests (one file per domain)
- `crates/slate-core/tests/links_roundtrip.rs` — property-based roundtrip test (proptest)
- `crates/slate-core/benches/` — Criterion benchmarks

## Naming Conventions

**Swift Files:**
- Views: `<Surface>View.swift` (e.g. `NoteEditorView.swift`, `TasksReviewView.swift`)
- Panels: `<Domain>Panel.swift` (e.g. `BacklinksPanel.swift`, `CitationsPanel.swift`)
- Sheets: `<Action>Sheet.swift` (e.g. `AddPropertySheet.swift`, `BulkRenameSheet.swift`)
- Stores: `<Domain>Store.swift` (e.g. `RecentVaultsStore.swift`, `PreferencesStore.swift`)
- Models: `<Surface>Model.swift` (e.g. `CommandPaletteModel.swift`)
- Extensions / helpers: `<Category>Extensions.swift` or `<Domain>Prefs.swift`
- Generated: `slate_uniffi.swift` (underscore, not CamelCase — uniffi convention)

**Rust Files:**
- Domain parsers: `<domain>.rs` (e.g. `links.rs`, `tasks.rs`)
- DB query modules: `<domain>_db.rs` (e.g. `links_db.rs`, `tasks_db.rs`)
- Migrations: `NNN_<description>.sql` (zero-padded three-digit number)
- Tests directory: `session/tests/<domain>.rs` — one file per test domain

**Swift Types:**
- Views: `PascalCase` structs conforming to `View`
- State/models: `PascalCase` classes, `@MainActor final class`
- Enums: `PascalCase` with `camelCase` cases
- Command IDs: `slate.<section>.<verb>` string constants in `SlateCommandID`

**Rust Types:**
- Public structs/enums: `PascalCase`
- Functions/methods: `snake_case`
- Module files: `snake_case.rs`

## Where to Add New Code

**New SwiftUI Panel (sidebar pane for a domain):**
- Implementation: `apps/slate-mac/Sources/SlateMac/<Domain>Panel.swift`
- State: Add `@Published` properties and async load methods to `AppState.swift`
- Tests: `apps/slate-mac/Tests/SlateMacTests/<Domain>PanelTests.swift`
- Wire into `FileListSidebar.swift` (left sidebar) or `MainSplitView.swift` (right sidebar tab)

**New Content Pipeline (new content type with AT representation):**
- Rust parser: `crates/slate-core/src/<domain>.rs` (pure parse function)
- Rust DB module (if indexed): `crates/slate-core/src/<domain>_db.rs`
- Add to `session.rs` as a `get_<domain>_blocks` method on `VaultSession`
- FFI binding: add method to `VaultSession` impl in `crates/slate-uniffi/src/lib.rs`
- AppState: add `currentNote<Domain>Blocks`, `isLoading<Domain>Blocks`, load task handle, `load<Domain>Blocks` async method
- View: `apps/slate-mac/Sources/SlateMac/<Domain>View.swift`
- Tests: `apps/slate-mac/Tests/SlateMacTests/Milestone<X>IntegrationTests.swift`

**New Command Palette Command:**
- Add constant to `SlateCommandID` in `apps/slate-mac/Sources/SlateMac/SlateCommands.swift`
- Append to `SlateCommandID.all` array (drift test enforces coverage)
- Register in `registerCoreCommands` in `SlateCommands.swift` with section, title, shortcut, action
- Tests: `apps/slate-mac/Tests/SlateMacTests/SlateCommandsTests.swift` (drift test validates registration)

**New SQLite Migration:**
- Add `NNN_<description>.sql` to `crates/slate-core/migrations/` (next number in sequence)
- Add `Migration { description: "...", sql: include_str!("../migrations/NNN_...sql") }` entry to `MIGRATIONS` array in `crates/slate-core/src/db.rs`
- Never reorder or modify existing migrations

**New Preference:**
- Add field to `MathPrefs`, `CodePrefs`, or create a new `<Domain>Prefs.swift` struct
- Add `load<Domain>Prefs` / `save<Domain>Prefs` methods to `PreferencesStore.swift`
- Add `static let <domain>Key = "slate.prefs.<domain>"` key constant

**New Rust Core Module:**
- Create `crates/slate-core/src/<domain>.rs`
- Declare as `pub mod <domain>;` in `crates/slate-core/src/lib.rs`
- Re-export public types from `lib.rs` (`pub use <domain>::...`)

## Special Directories

**`apps/slate-mac/Sources/slate_uniffiFFI/`:**
- Purpose: SwiftPM systemLibrary pointing at the generated C header
- Generated: `slate_uniffiFFI.h` is generated; `module.modulemap` is committed
- Do not edit the header; run `uniffi-bindgen` via `build-mac-app.sh`

**`.slate/` (inside a user vault at runtime):**
- Purpose: Hidden vault metadata directory created by `VaultSession.open`
- Contains: `cache.sqlite` (index), `prefs.json` (citations prefs), `tmp/` (atomic write staging)
- Not in the repository — created at runtime inside the user's chosen vault folder

**`graphify-out/`:**
- Purpose: Output cache for the graphify skill's AST/semantic analysis
- Generated: Yes
- Committed: No (in .gitignore)

**`target/`:**
- Purpose: Cargo build output including compiled dylib/staticlib
- Generated: Yes
- Committed: No

**`.planning/codebase/`:**
- Purpose: GSD codebase map documents consumed by /gsd:plan-phase and /gsd:execute-phase
- Generated: Yes (by /gsd:map-codebase)
- Committed: Yes
