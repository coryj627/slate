# 14 — Localization (l10n): Inventory + Strategy

**Status:** 📝 Strategy note (2026-07-06). Not scheduled — no milestone owns execution. Umbrella issue: [#642](https://github.com/coryj627/slate/issues/642); narrow precursor audit: [#264](https://github.com/coryj627/slate/issues/264).
**Purpose:** One place per-feature specs can point instead of re-noting "English-only" each time, plus the settled *shape* of an eventual localization pass so today's copy decisions don't paint us into a corner.

## Current state

Every user-facing string in Slate is English-only, and much of it is **normative spec copy** — exact sentences pinned by executable specs and tests (e.g. the sync-detection recommendation table in `09_sync_cli/m_spec.md` §M-1, `SyncDetectionReport.audio_summary`, `QueryResultSet.summary`). That is deliberate: the accessible-first architecture pre-renders speech-ready strings in core so every host platform announces identically, and spec-pinned copy is testable copy.

## Inventory — where user-facing copy lives

| Layer | Copy | Examples | Volume |
|---|---|---|---|
| `slate-core` (Rust) | Pre-rendered summaries + normative recommendation/warning copy crossing the FFI | `audio_summary`, `multi_sync_warning`, recommendation sentences, `QueryResultSet.summary`, math speech via MathCAT, `VaultError` `Display` texts | Moderate, high-stakes (spec-pinned, AT-facing) |
| `slate-uniffi` | Mirrors carry core strings through, **plus** its own English `#[error("…")]` `Display` templates on the FFI `VaultError`/`CommandError` enums (`io error: …`, `invalid vault-relative path …`, `unknown command id: …`) | `VaultError`/`CommandError` `#[error]` messages (`lib.rs`) | Small, but real — surfaces via Swift `localizedDescription` |
| `apps/slate-mac` (Swift) | Panel labels, empty states, menu items, command palette labels/hints, AX labels/announcement templates, sheet copy | `LeafEmptyState` messages, `SlateCommandID` labels/hints, `"\(title) panel."` announcements, Settings | Large, chrome-heavy |
| `slate-cli` | `--help` text (clap), human-format output lines, `slate: `-prefixed errors/warnings | `"not a vault directory"`, `"vault cache is busy…"`, unfilled-prompt warnings | Small, script-adjacent (see caveat below) |
| Docs / templates | Help guide, runbooks, template prompt labels in user vaults | `docs/help/…` | Out of scope for app l10n; user content is never localized |

Special cases worth naming now:
- **MathCAT speech** (Milestone K) already has its own language rule files upstream — math speech localizes through MathCAT's mechanism, not ours.
- **CLI caveat:** human-format *data* lines and `--format json|tsv` output are a stability contract (`slate.cli.v1`) and scripts parse stderr messages; localized CLI output would break scripts. Decision below exempts the CLI data/error contract from l10n (help text may localize; contract strings do not).
- **Dates:** the CLI deliberately renders authored UTC calendar dates (`YYYY-MM-DD`, never tz-shifted); locale date formats apply only to *presentation* surfaces (Mac app), never to contract output.

## Strategy decisions (the shape, decided now; execution unscheduled)

1. **Core-side catalogs for core-born strings.** When l10n is scheduled, `slate-core` gains a locale input (session-config or per-call) and localizes its own pre-rendered strings via compiled-in catalogs (Fluent is the default candidate; a plain per-locale table is acceptable at our string volume). Rationale: pre-rendered accessible representations are a locked convention (`05` §1.2 "One canonical structure, many accessible representations"; the math pipeline that produces `speech` lives in §6.2), and every future host (Windows Milestone W, iOS) inherits one catalog instead of re-implementing sentence assembly per platform — sentence *grammar* (pluralization, list joining, e.g. "3 sync systems detected: …") must not be reassembled host-side per language. **Structured `Display` error texts are the exception:** `VaultError`/`CommandError` `#[error("…")]` messages (in both `slate-core` and `slate-uniffi`) are typed enum variants with a stable discriminant, so hosts localize them by mapping the *variant* to host-owned prose rather than rendering the English `Display`/`localizedDescription` — no core catalog needed for these.
2. **Host-side String Catalogs for host chrome.** Swift labels/menus/announcement templates localize with Xcode String Catalogs (`.xcstrings`), independently of core. #264 (closed) already established the precursor hygiene pattern on the SettingsView footer strings: hoisted to `static let` and routed through `String(localized:)` (catalog-ready; ships English-only until a `.xcstrings` table exists), guarded by `SettingsLocalizationTests` asserting the copy round-trips unchanged. Extend that pattern — named constants routed through `String(localized:)`, not inline literals — as surfaces get touched.
3. **Normative copy stays normative.** Specs pin the *English source* string; translations are derived artifacts keyed by stable ids. Tests keep asserting the English source (catalog-key indirection must not weaken spec pinning); per-locale QA is a translation-pipeline concern, not a unit-test concern.
4. **The CLI contract does not localize.** `slate.cli.v1` json/tsv fields, wire slugs, exit-code messages that scripts match on, and human data lines stay English. clap `--help` text may localize if clap grows that cheaply; low priority.
5. **No speculative extraction.** Until a milestone owns l10n, we do NOT wrap strings in indirection "to prepare" — that's churn without a consumer. The only standing rules: (a) new Swift surfaces prefer named string constants over scattered inline literals when a file already collects them; (b) core strings that cross the FFI stay pre-rendered whole sentences (never host-assembled fragments), which is exactly what makes decision 1 executable later.

## Sequencing sketch (when scheduled)

1. #264-style hygiene sweep (Swift constants) — mechanical, PR-sized.
2. Core catalog spike: one feature's strings (sync detection is the ideal pilot — small, spec-pinned, AT-critical) through Fluent with a locale-in-session-config; measure binary-size and FFI impact.
3. String Catalogs for Mac chrome; VoiceOver pass per locale (the real cost — localized AT copy needs native-speaker AT review, not just translation).
4. CLI help text last, if at all.

## Non-goals

- Localizing user vault content, template bodies, or anything the user authored.
- RTL/layout mirroring work before a target language demands it.
- Scheduling: this note creates no milestone; #642 stays open as the umbrella.
