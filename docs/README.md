# Accessible Obsidian-Compatible Flutter App — Planning Pack

This folder contains separate Markdown artifacts generated from the Obsidian accessibility/replacement-app research conversation.

## Files

1. `01_detailed_roadmap.md` — Product roadmap, phases, principles, success metrics, Bases/Graph/Sync direction.
2. `02_phase_0_plan.md` — Discovery, validation, clean-room, parser, Bases, Graph, plugin corpus, LiveSync, and accessibility spike plan.
3. `03_phase_1_plan.md` — Accessible vault MVP implementation plan.
4. `04_draft_architecture_plan.md` — Draft technical architecture for vault, metadata, Bases, Graph, plugins, sync, accessibility, persistence, and security.
5. `05_public_conversation_and_research_transcript.md` — Public transcript/research log. This does not include private chain-of-thought.

## Notes

The documents use a shared source register rather than repeating long inline citations in every paragraph. They are intended as planning artifacts, not legal advice or final engineering specifications.
